# Chart Patterns

Charts are in scope only when a static SVG chart communicates better than a diagram.

## Use Charts For

- simple comparisons across categories
- one trend over time
- one small grouped comparison

Do not force a chart when a comparison card or annotated diagram would be clearer.

## Preferred Chart Types

### Bar chart

Best for:

- discrete categories
- magnitude comparison

Rules:

- horizontal bars for long labels
- vertical bars only when labels are short
- label bars directly when space allows

### Line chart

Best for:

- one or two time-series trends

Rules:

- keep the axis simple
- avoid more than two series in chat-sized charts
- highlight the key turning point with an annotation instead of a legend-heavy layout

### Small metric strip

Best for:

- 2 to 4 headline values with small captions

Rules:

- use equal-sized cards
- keep numbers rounded
- avoid dashboard clutter

## Styling Rules

- flat fills only
- no stacked legend complexity unless essential
- labels should be readable without zooming
- use one accent color plus neutral structure
